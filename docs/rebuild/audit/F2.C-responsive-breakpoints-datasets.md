# F2.C - Responsive breakpoints + dataset policy

Fecha: 2026-04-23

## Objetivo

Cerrar el drift detectado en F0.4/F2.A entre shell family JS y shell spacing CSS, y limpiar solo el dataset muerto confirmado sin romper compatibilidad legacy.

## Cambios realizados

### 1. Breakpoints JS alineados

Archivo:

- `frontend/src/hooks/useResponsiveLayout.js`

Antes:

- `<= 640` -> `phone`
- `<= 820` -> `tablet-compact`
- `<= 1024` -> `tablet`
- `<= 1280` -> `laptop`

Ahora:

- `<= 767` -> `phone`
- `<= 820` -> `tablet-compact`
- `<= 1023` -> `tablet`
- `<= 1280` -> `laptop`

Esto deja los cortes macro en:

- mobile: hasta 767px
- tablet: 768px a 1023px
- desktop: desde 1024px

### 2. Shell spacing CSS alineado

Archivo:

- `frontend/src/styles/tailwind.css`

El spacing desktop de shell paso de:

```css
@media (min-width: 80rem)
```

a:

```css
@media (min-width: 64rem)
```

El spacing tablet se mantiene en `48rem` (768px). Con esto, la capa CSS de shell spacing se alinea con la familia macro que decide JS.

### 3. Dataset muerto eliminado

Archivo:

- `frontend/src/hooks/useResponsiveLayout.js`

Se elimino la escritura y limpieza de:

- `document.body.dataset.shell`

Verificacion posterior:

- `body[data-shell]`: 0 consumidores.
- `dataset.shell`: 0 escrituras/deletes.

### 4. Datasets conservados por compatibilidad

Se conservaron:

- `body.dataset.theme`: necesario para tema visual.
- `body.dataset.viewport`: sigue consumido por `app.css`.
- `body.dataset.inputMode`: sigue consumido por `app.css` para targets touch y controles.

No se retiro compatibilidad legacy en esta subfase.
