# F3.C - Survey primitives + a11y pass

Fecha: 2026-04-23

## Objetivo

Cerrar el pase minimo de primitives y accesibilidad de `SurveyView.jsx` detectado en F3.A, sin cambiar scoring, hooks, backend, flujo de resultados ni layout/patterns de F3.D.

## Cambios realizados

### 1. Opciones de Survey migradas a primitives de foundation

Archivo:

- `frontend/src/components/SurveyView.jsx`

Se retiro el helper local `SurveyChoiceCard`, que combinaba:

- `input className="sr-only"`;
- `SurfaceCard interactive`;
- shadow arbitrario para selected state;
- semantica local por opcion.

Ahora las preguntas usan primitives reales:

- `Radio` para preguntas `single`;
- `Checkbox` para preguntas `multi`.

Esto deja la interaccion de opciones en la capa foundation, con focus visible, estados `checked`, `disabled`, `invalid`, hint y `aria-describedby` propagados al control nativo.

### 2. Agrupacion semantica por pregunta

Archivo:

- `frontend/src/components/SurveyView.jsx`

Las preguntas `single` y `multi` ahora renderizan:

- `fieldset`;
- `legend` oculto con `sr-only`;
- `aria-describedby` compartido;
- `aria-invalid` cuando existe error de validacion;
- `aria-required="true"`.

Esto evita que una lista de opciones quede expuesta como controles aislados sin nombre de grupo.

### 3. Errores y ayudas conectados a controles

Archivo:

- `frontend/src/components/SurveyView.jsx`

Se agregaron ids estables por pregunta:

- `survey-question-<id>-description`;
- `survey-question-<id>-flow-error`;
- `survey-question-<id>-validation-error`.

El helper `mergeDescribedBy` compone los ids disponibles y los pasa a:

- `Radio`;
- `Checkbox`;
- `Select`;
- `TextArea`;
- `fieldset`.

Los `InlineMessage` de error de flujo y validacion ahora tienen id, por lo que los controles quedan enlazados al texto que explica el problema.

### 4. Select y TextArea alineados con a11y basica

Archivo:

- `frontend/src/components/SurveyView.jsx`

Las preguntas `select` y `text` ahora incluyen:

- `id`;
- `name`;
- `required`;
- `invalid`;
- `aria-describedby`.

Se conserva `Field` como wrapper de label para no redisenar el formulario en esta subfase.

### 5. Payload multi-select preservado

Archivo:

- `frontend/src/components/SurveyView.jsx`

Se agrego `buildNextMultiAnswer` para construir respuestas multiples respetando el orden original de `question.options`.

La llamada a `onAnswerChange(question.id, nextAnswer)` se mantiene intacta para no tocar el contrato de `useAssessmentFlow`.

### 6. Guard source-level agregado

Archivo:

- `test/f3-auth-survey-smoke.test.js`

Cobertura agregada:

- `SurveyView` importa y usa `Checkbox` y `Radio`;
- `SurveyChoiceCard` ya no existe;
- ya no queda `input className="sr-only"` local;
- ya no queda el shadow arbitrario de selected choice;
- `fieldset` queda conectado a `aria-describedby`, `aria-invalid` y `aria-required`;
- `legend` usa el titulo de la pregunta;
- `InlineMessage` de flujo y validacion tiene ids;
- `Select` y `TextArea` tienen id, required, invalid y `aria-describedby`.
