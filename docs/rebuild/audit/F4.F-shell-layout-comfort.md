# F4.F - Shell / layout comfort pass

Fecha: 2026-04-23

## Objetivo

Cerrar el pase de F4 dedicado a proporciones, simetria, aire y comodidad por shell dentro de `CoursesView.jsx`.

F4.F se limito a:

- usar el `JourneyStepper` que ya se calculaba, sin crear otro flujo;
- marcar explicitamente el modo de layout de ruta por shell;
- dar mas ancho al detalle central en desktop;
- reducir rails laterales que estaban ocupando demasiado;
- evitar splits internos apretados en tablet/laptop;
- hacer que ajustes no colapsen tres selects en una fila estrecha;
- mantener fuera lesson shell, activity chrome, simulaciones, services, hooks y dominio.

## Cambios aplicados

### DashboardSceneBar / JourneyStepper

- `DashboardSceneBar` ahora recibe `journeySteps`.
- Se agrego `hasJourneySteps` para renderizar el stepper solo si hay pasos disponibles.
- El `JourneyStepper` vive como contexto compacto dentro de la barra de escenas, marcado con `data-sd-journey-stepper="courses-route"`.
- El stepper usa `compact={shellFamily !== 'desktop'}` para evitar saturar mobile/tablet.
- Con esto se cierra la deuda de `journeySteps` calculado pero no usado.

### Route layout por shell

- Se agrego `routeLayoutMode`:
  - `mobile-stack`;
  - `tablet-two-pane`;
  - `desktop-balanced`.
- La region de ruta expone `data-sd-route-layout={routeLayoutMode}` para que el modo de composicion quede verificable.
- Tablet baja a dos columnas mas comodas: rail de modulos mas contenido y detalle con mas peso.
- Desktop cambia a una grilla balanceada que reduce ambos rails y da mas espacio al detalle central.

### ModuleMissionBoard

- El detalle del modulo usa gaps basados en `--sd-shell-pane-gap`.
- El split interno se reserva solo para `2xl`, evitando que tablet/laptop aprieten lista interna + rail de apertura.
- Se agrego `data-sd-module-layout` con:
  - `desktop-comfort`;
  - `stacked-comfort`.
- El rail de apertura dentro del detalle deja de ser sticky para evitar comportamiento incomodo en layouts apilados.

### ProgressScene

- El layout de progreso usa `--sd-shell-pane-gap`.
- Desktop/tablet dan mas espacio al bloque principal de competencias y reducen el rail lateral.

### SettingsScene

- El split hero/controles se balanceo para dar mas espacio a controles.
- Los selects de preferencias pasan de `md:grid-cols-3` a `md:grid-cols-2 xl:grid-cols-3`.
- Esto evita tres controles apretados en tablet y mantiene tres columnas solo cuando hay ancho suficiente.

## Guardas agregadas

Se amplio `test/f4-courses-dashboard-smoke.test.js` con guardas F4.F:

- `DashboardSceneBar` debe renderizar `JourneyStepper` con `journeySteps`.
- `CoursesView` debe exponer `routeLayoutMode`.
- La ruta debe marcar `data-sd-route-layout`.
- Tablet debe usar una grilla `18rem-20rem` + detalle flexible.
- Desktop debe usar una grilla con rails mas contenidos y centro `1.55fr`.
- `ModuleMissionBoard` no debe recuperar el split tablet/laptop anterior.
- `ModuleMissionBoard` no debe recuperar `sticky={shellFamily === 'desktop'}` en el rail interno.
- `SettingsScene` no debe recuperar `md:grid-cols-3` para los selects.
