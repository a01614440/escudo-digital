## Bugs/deuda viva confirmada

### 1. DeviceShell render-as-function

`frontend/src/shells/DeviceShell.jsx` todavia hace:

```js
const renderShell = SHELL_RENDERERS[shellFamily] || SHELL_RENDERERS.desktop;
return renderShell({ ... });
```

Esto funciona hoy porque los shells no tienen hooks, pero es deuda tecnica real: si un shell agrega hooks despues, se puede romper el orden de hooks al cambiar `shellFamily`.

### 2. Breakpoint drift JS/CSS

`useResponsiveLayout.js` usa estos cortes:

- `<= 640` -> `phone`
- `<= 820` -> `tablet-compact`
- `<= 1024` -> `tablet`
- `<= 1280` -> `laptop`
- `> 1280` -> `desktop`

`tailwind.css` usa:

- `@media (min-width: 48rem)` = 768px
- `@media (min-width: 80rem)` = 1280px

Sigue existiendo drift entre shell JS y spacing CSS:

- 641-767px: JS ya usa `TabletShell`, CSS conserva spacing base movil.
- 1025-1279px: JS ya usa `DesktopShell`, CSS conserva spacing tablet.

### 3. body.dataset policy

Conteo actual:

- `body[data-viewport]`: 65 matches.
- `body[data-shell]`: 0 matches.
- `body[data-input-mode]`: 11 matches.
- `body[data-theme]`: 421 matches.
- `dataset.viewport`: 2 writes/deletes.
- `dataset.shell`: 2 writes/deletes.
- `dataset.inputMode`: 2 writes/deletes.
- `dataset.theme`: 1 write.

Politica recomendada:

- Mantener `theme`: lo usa la capa visual.
- Mantener `viewport`: sigue vivo por `app.css`; retirarlo pertenece a F8/cleanup legacy.
- Mantener `inputMode`: `app.css` lo usa para targets touch y controles.
- Eliminar `shell`: dead write confirmado.

### 4. Route intent immersive sin comportamiento especial

`navigationPolicy.js` define `lesson.shellIntent = 'immersive'`, y los shells lo escriben en `data-shell-intent`.

Pero el comportamiento real solo distingue parcialmente `focus` vs no-`focus`. `immersive` todavia no tiene tratamiento especifico. No es bloqueo para F2 closeout, pero debe quedar documentado para F5/F6.

### 5. Tests de shell son contract/source-level, no render-level

`shell-architecture.test.js` y `responsive-smoke.test.js` cubren:

- responsive profile
- shell family
- input mode
- route metadata
- navigation model
- shell policies

No cubren render real de `DeviceShell`, slots ni el antipattern `renderShell({...})`. F2 debe agregar proteccion minima.

## Alcance exacto recomendado para F2 closeout

- `test/shell-architecture.test.js`
- docs de F2

- No quitar `viewport` todavia.
- No quitar `inputMode` todavia.
