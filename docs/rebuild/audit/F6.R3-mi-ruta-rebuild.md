# F6.R3 - Mi ruta rebuild

Fecha: 2026-04-23
Estado: cerrada localmente

## Objetivo

Rearmar `Mi ruta` para que deje de sentirse como varias superficies hermanas pegadas y vuelva a una lectura mas clara:

- continuidad visible;
- CTA principal dominante;
- tabs integradas en el mismo shelf;
- menos fragmentacion en el primer pliegue;
- body `detail-first`, con el rail subordinado.

## Cambios implementados

### 1. Shelf superior unificado

Archivo: `frontend/src/components/CoursesView.jsx`

- `RouteBriefing` ahora absorbe la consola de tabs mediante `DashboardSceneBar`.
- El shelf superior expone `data-sd-route-shelf="integrated"`.
- La continuidad ya no se lee como hero + tabs + body separados; ahora vive en una sola superficie inversa.
- Se agrego un bloque corto de `Siguiente paso` dentro del mismo shelf para unir continuidad y CTA.
- El progreso total de la ruta se mantiene dentro del mismo shelf en vez de quedar como bloque hermano aparte.

### 2. Consola de tabs integrada

Archivo: `frontend/src/components/CoursesView.jsx`

- `DashboardSceneBar` deja de renderizar una `SurfaceCard` propia.
- Ahora funciona como consola embebida (`data-sd-route-console="integrated"`).
- La copia del tab activo vive dentro del mismo lenguaje visual inverso del shelf principal.
- `DashboardTabs` soporta `tone="inverse"` para que el cambio de tab no se vea lavado dentro de la cabina superior.

### 3. Layout de ruta `detail-first`

Archivo: `frontend/src/components/CoursesView.jsx`

- `CoursesView` ya no usa el modo previo `tablet-two-pane` / `desktop-two-pane`.
- La ruta ahora expone:
  - `mobile-stack`
  - `tablet-stack`
  - `desktop-detail-first`
- En desktop, el body pasa a `xl:grid-cols-[minmax(0,1.18fr)_minmax(16rem,18rem)]`.
- `ModuleMissionBoard` se renderiza antes que `RouteNavigatorRail`.
- `data-sd-route-comfort` ahora expresa `detail-first` o `stacked`.

### 4. Rail subordinado y pills menos apretadas

Archivo: `frontend/src/components/CoursesView.jsx`

- `RouteNavigatorRail` pasa a `data-sd-route-rail="secondary"`.
- El copy del rail se acorta a `Explora tu ruta` / `Cambia de modulo sin perder el siguiente paso.`
- El filtro de nivel deja de usar una sub-card adicional y se resuelve como un bloque interno mas ligero.
- `RouteModulePill` deja el layout `grid-cols-[auto_1fr_auto]` y pasa a una composicion mas flexible para reducir wrapping torpe y tension visual.
