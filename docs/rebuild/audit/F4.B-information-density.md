# F4.B - Information architecture / density pass

Fecha: 2026-04-23

## Objetivo

Cerrar el primer pase de implementacion de F4 sobre `CoursesView.jsx` sin redisenar la ruta completa.

F4.B se limito a bajar densidad informativa, reducir copy redundante, quitar badges/stats secundarios y aclarar jerarquia. No se trabajo CTA profundo, layout comfort por shell, detalle fino de modulo ni lesson shell.

## Cambios por zona

### Hero / empty states

- Se acorto el copy de generacion de ruta.
- Se simplifico la meta superior a una lectura mas directa: diagnostico primero, continuidad despues.
- Se redujo el copy de acciones para evitar explicar demasiado lo que ya comunica el layout.

### RouteHero

- Se elimino el stat de `Score total` de la cabecera de ruta.
- Se retiro `Ultimo acceso` del hero principal.
- Se retiro el badge de nivel del assessment.
- El hero conserva solo dos senales: avance de ruta y foco/gap.

Esto deja la cabecera menos saturada y evita competir con `ProgressScene`.

### ContinuityConsole

- Se elimino metadata secundaria de modulo.
- Se retiro el `KeyValueBlock` de categoria/tiempo.
- Se acorto el hint del progreso.
- Se retiro el override local `!grid-cols-1` sobre `ProgressSummary`.

La consola queda centrada en una sola pregunta: que sigue ahora.

### TopSupportBand

- Se redujo de una composicion doble a un solo `SupportRail`.
- Se retiro el `ProgressSummary` duplicado del foco/gap.
- Se mantienen solo dos items de `quickGuide`.

El foco/gap ya vive en `RouteHero` y en `RouteInsightRail`, por lo que este bloque ya no repite informacion.

### DashboardSceneBar

- Se retiro el badge `x/y modulos cerrados`.
- La barra queda como selector de escena y contexto breve, no como otro panel de progreso.

### RouteNavigatorRail / RouteModulePill

- Se retiro el contador de visibles del header.
- Se acorto copy del navegador y del filtro por nivel.
- Cada pill dejo de mostrar contador de actividades y duracion.
- Cada pill conserva estado y badge `Siguiente` cuando aplica.

Esto baja ruido en el rail sin cambiar seleccion, bloqueo ni orden.

### ModuleMissionBoard

- Se retiro el stat de tiempo del `StatStrip`.
- Se retiro el texto extra debajo del hero.
- Se retiro la metrica de visitas.
- Se retiro el override local `!grid-cols-1`.
- Se simplifico el copy de la ruta interna.

El detalle queda menos cargado, pero el pase fino lista <-> detalle queda reservado para F4.D.

### RouteInsightRail

- Se retiro la lectura duplicada de fortaleza.
- Se retiro el mensaje local del modulo seleccionado.
- Se retiro la card extra de guia.
- El rail conserva progreso total, gap, siguiente desbloqueo y estado admin.

Esto evita que el costado repita lo que ya aparece en el hero, navegador y detalle.

### ProgressScene

- Se retiro `Ultimo acceso` del `StatStrip`.
- Se retiraron badges de fortaleza/gap del hero.
- Se acorto el titulo principal.

La reorganizacion profunda de progreso/stats queda para F4.E.

### SettingsScene

- Se retiro `duration` del `settingsStrip`.
- Se acortaron titulo, subtitulo y meta del hero.
- Se simplifico el subtitulo de preferencias.

Los controles siguen intactos; solo se redujo la cantidad de resumen superior.

## Guardas agregadas

Se agrego `test/f4-courses-dashboard-smoke.test.js` con guardas de fuente para evitar regresion de densidad:

- `RouteHero` no debe recuperar `Score total`, `Ultimo acceso` ni badge de nivel.
- `ContinuityConsole` no debe recuperar metadata de categoria/tiempo ni override local.
- `TopSupportBand` no debe duplicar `ProgressSummary`.
- `DashboardSceneBar` no debe recuperar contador de modulos.
- `RouteModulePill` no debe recuperar duracion ni contador de actividades.
- `RouteNavigatorRail` no debe recuperar contador de visibles.
- `ModuleMissionBoard` no debe recuperar tiempo, visitas ni override local.
- `RouteInsightRail` no debe recuperar `quickGuide`, fortaleza duplicada ni mensaje de modulo seleccionado.
- `ProgressScene` y `SettingsScene` no deben recuperar los stats de menor utilidad en hero.
