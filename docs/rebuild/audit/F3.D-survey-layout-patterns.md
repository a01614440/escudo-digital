# F3.D - Survey layout / patterns pass

Fecha: 2026-04-23

## Objetivo

Cerrar el pase minimo de layout y patterns de `SurveyView.jsx` despues de F3.C, adoptando los patterns de foundation donde reducen deuda real:

- `QuestionPage` para la pregunta activa;
- `AssessmentLayout` para la escena activa de encuesta;
- `InfoPanel` para la ayuda contextual.

No se cambio intro/loading/results, no se cambio flow, no se toco scoring y no se tocaron hooks de dominio.

## Cambios realizados

### 1. QuestionBoard delega en QuestionPage

Archivos:

- `frontend/src/components/SurveyView.jsx`
- `frontend/src/patterns/QuestionPage.jsx`

`QuestionBoard` ya no arma manualmente:

- `fieldset`;
- `legend`;
- controles por tipo;
- `Select`;
- `TextArea`;
- mensaje de error de validacion.

Ahora usa `QuestionPage` con:

- `id` estable por pregunta;
- `type={question.type}`;
- `name={question.id}`;
- `options` normalizadas de `helper` a `hint`;
- `value` controlado;
- `onValueChange={(nextValue) => onAnswerChange(question.id, nextValue)}`;
- `required`;
- `error`;
- `errorId`;
- `errorTitle`;
- `aria-describedby` para enlazar error de flujo cuando existe.

Esto mantiene el contrato de respuestas de `SurveyView` sin tocar `useAssessmentFlow`.

### 2. QuestionPage preserva accesibilidad ganada en F3.C

Archivo:

- `frontend/src/patterns/QuestionPage.jsx`

Se agrego soporte minimo para:

- `aria-describedby` externo;
- `errorId` configurable;
- `errorTitle` configurable;
- `aria-required` en `fieldset` cuando `required` es true.

Esto permite adoptar `QuestionPage` sin perder:

- conexion con `flowError`;
- id estable para validation error;
- semantica requerida del grupo de opciones.

### 3. SurveyStageScene usa AssessmentLayout

Archivo:

- `frontend/src/components/SurveyView.jsx`

La escena activa de preguntas dejo de usar `WorkspaceLayout` con grid local desktop:

- `xl:grid-cols-[minmax(16.5rem,17.5rem)_minmax(0,1.28fr)_minmax(18.5rem,19.5rem)]`
- `2xl:grid-cols-[minmax(17rem,18rem)_minmax(0,1.35fr)_minmax(19rem,20rem)]`

Ahora usa `AssessmentLayout` con slots:

- `hero`;
- `progress`;
- `question`;
- `insight`.

Esto mueve la composicion de Survey hacia el layout de dominio creado en F1 patterns.

### 4. Ayuda contextual usa InfoPanel

Archivos:

- `frontend/src/components/SurveyView.jsx`
- `frontend/src/patterns/InfoPanel.jsx`

`SurveyInsightDeck` ahora usa `InfoPanel` con `tone="coach"` e items educativos.

Se agrego prop `as` a `InfoPanel`, con default `aside`, para poder renderizarlo como `div` dentro del slot `insight` de `AssessmentLayout` sin crear un `aside` anidado dentro de otro `aside`.

### 5. Overrides locales reducidos en la escena activa

Archivo:

- `frontend/src/components/SurveyView.jsx`

En `SurveyCommandDeck` se retiraron overrides `!grid-cols-*` de:

- `ProgressSummary`;
- `JourneyStepper`.

La escena activa ya depende de los defaults de foundation y de `AssessmentLayout`.
