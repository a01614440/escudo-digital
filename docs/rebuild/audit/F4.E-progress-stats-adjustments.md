# F4.E - Progress / stats / adjustments pass

Fecha: 2026-04-23

## Objetivo

Cerrar el pase de F4 dedicado a progreso, estadisticas utiles y ajustes/regeneracion dentro de `CoursesView.jsx`.

F4.E se limito a:

- priorizar stats accionables;
- quitar lecturas decorativas del resumen de progreso;
- hacer que snapshots no ocupen espacio si no hay historial real;
- convertir el rail de progreso en una lectura de siguiente foco;
- hacer que ajustes se lean como controles secundarios seguros;
- dejar regeneracion con CTA explicito sin tocar services, hooks ni dominio.

- `frontend/src/components/CoursesView.jsx`

## Cambios aplicados

### ProgressScene

- Se agrego `progressStrip` como resumen util de progreso:
  - avance de ruta;
  - fortaleza principal;
  - gap principal.
- Se retiro `key: 'shield'` del hero de progreso.
- El titulo del hero paso a `Progreso util de tu ruta`.
- La meta del hero ahora aclara que se muestran avance, fortaleza y gap, sin datos decorativos.
- La seccion de competencias mantiene las barras por tema, pero con copy mas directo.
- La seccion de historial ahora es condicional: solo aparece si `history.length` tiene snapshots reales.
- Se elimino el empty-state visible `Sin snapshots todavia` para no ocupar espacio con datos inexistentes.
- La lectura lateral cambio de fortaleza a `Siguiente foco`, usando `weakestTopic` como dato accionable.
- El rail lateral paso a `Senales de aprendizaje`, centrado en errores frecuentes y senales fuertes.
- El item `key: 'prefs'` ya no existe como mezcla de preferencias dentro de progreso.

### SettingsScene

- Se agrego `selectedTopicCount` para resumir temas priorizados.
- `settingsStrip` ahora resume:
  - estilo;
  - dificultad;
  - temas priorizados.
- El hero de ajustes cambio a `tone="support"` para sentirse secundario frente a la ruta.
- El titulo paso a `Ajustes de ruta sin perder progreso`.
- La meta explicita que no se toca diagnostico, scoring ni avance guardado.
- La card principal bajo de `variant="spotlight"` a `variant="panel"`.
- El copy de preferencias aclara que solo cambia presentacion/prioridad visible.
- El hint de temas priorizados se acorto y se centro en reforzar prioridades concretas.
- El CTA de regeneracion cambio a `Button variant="primary"` con:
  - `data-sd-settings-cta="courses-regenerate"`;
  - `aria-label="Actualizar ruta con estas preferencias"`.
- El mensaje de seguridad ahora dice `Se conserva tu base`.

## Guardas agregadas

Se amplio `test/f4-courses-dashboard-smoke.test.js` con guardas F4.E:

- `ProgressScene` debe usar `progressStrip`.
- `ProgressScene` debe conservar `route`, `focus` y `gap` como stats principales.
- `ProgressScene` no debe recuperar `key: 'shield'` ni `Shield`.
- Snapshots deben ser condicionales y no debe volver `Sin snapshots todavia`.
- La lectura lateral debe usar `weakestTopic` y `tone="warning"`.
- `SettingsScene` debe tener `selectedTopicCount`, `key: 'topics'`, `tone="support"`, `variant="panel"` y CTA de regeneracion marcado.
- `SettingsScene` no debe recuperar `variant="hero"` para el CTA de regeneracion.
