# F3.F - Results / perfil / CTA closeout

Fecha: 2026-04-23

## Objetivo

Cerrar el resultado de Survey como salida clara hacia la ruta:

- perfil/riesgo visible;
- recomendaciones legibles;
- CTA `Ver mi ruta` dominante;
- fallback claro para `courseError`;
- continuidad hacia Courses sin tocar wiring, hooks, services ni backend.

No se cambio `resolveActiveRoute.jsx`, `useAssessmentFlow.js`, `course.openCourses`, scoring, services ni backend.

## Cambios realizados

### 1. ResultsScene usa layout de dominio

Archivo:

- `frontend/src/components/SurveyView.jsx`

`ResultsScene` dejo de usar `WorkspaceLayout` con grid local desktop:

- `xl:grid-cols-[minmax(16.5rem,17.5rem)_minmax(0,1.22fr)_minmax(19rem,20rem)]`;
- `2xl:grid-cols-[minmax(17rem,18rem)_minmax(0,1.3fr)_minmax(19.5rem,20.5rem)]`.

Ahora usa `AssessmentLayout` con slots:

- `hero`;
- `progress`;
- `question`;
- `insight`.

Esto deja results alineado con la misma arquitectura de dominio que la escena activa de preguntas.

### 2. Perfil/riesgo usa tokens tipograficos

Archivo:

- `frontend/src/components/SurveyView.jsx`

El aside de perfil dentro de `SurveyStageHero` ya no usa:

- `font-display text-5xl`;
- `leading-[0.92]`;
- `tracking-[-0.06em]`.

Ahora usa:

- `rounded-[var(--sd-radius-lg)]`;
- `sd-title`;
- `sd-eyebrow`.

Esto evita tipografia ad hoc en el perfil de riesgo.

### 3. ResultsCommandDeck queda como perfil accionable

Archivo:

- `frontend/src/components/SurveyView.jsx`

`ResultsCommandDeck` ahora centraliza:

- `resultLevel`;
- `resultSummary`;
- `ProgressSummary` sin override local `!grid-cols-1`;
- `KeyValueBlock` con perfil y siguiente paso.

La lectura del perfil queda conectada directamente con la accion de abrir ruta.

### 4. Recomendaciones renderizadas como lista semantica

Archivo:

- `frontend/src/components/SurveyView.jsx`

`ResultsMainDeck` ahora renderiza recomendaciones con:

- `ol`;
- `li`;
- numeracion visual preservada.

Esto mejora la lectura estructural sin cambiar contenido ni datos de assessment.

### 5. CTA `Ver mi ruta` queda dominante y descrito

Archivo:

- `frontend/src/components/SurveyView.jsx`

`ResultsRouteRail` ya no deja el CTA solo como footer secundario del rail.

Ahora el rail contiene:

- descripcion enlazada con `id="survey-results-route-summary"`;
- `Button variant="primary" size="lg"`;
- `aria-describedby={ctaDescription}`;
- texto principal `Ver mi ruta`;
- fallback de error `Intentar abrir mi ruta de nuevo`;
- accion secundaria `Reiniciar encuesta`.

El CTA sigue llamando exactamente `onTakeCourses`.

### 6. Fallback de `courseError` enlazado al CTA

Archivo:

- `frontend/src/components/SurveyView.jsx`

Cuando existe `courseError`:

- `InlineMessage` usa `id="survey-results-route-error"`;
- el CTA agrega ese id a `aria-describedby`;
- el texto cambia a `Intentar abrir mi ruta de nuevo`.

Esto hace que el retry tenga contexto visible y accesible.

### 7. Wiring de handoff verificado sin tocarlo

Archivo revisado en modo lectura:

- `frontend/src/route-containers/resolveActiveRoute.jsx`

El wiring sigue siendo:

- `onTakeCourses: () => course.openCourses({ answers, assessment, authToken, adminAccess })`.

No se modifico porque no estaba roto.

### 8. Guards source-level agregados

Archivo:

- `test/f3-auth-survey-smoke.test.js`

Cobertura agregada:

- results usa `AssessmentLayout`;
- results ya no usa `WorkspaceLayout`;
- results ya no usa grid local desktop;
- perfil usa `sd-title` y no tracking negativo;
- recomendaciones usan `ol`;
- CTA usa `variant="primary"` y `size="lg"`;
- CTA mantiene `onClick={onTakeCourses}`;
- CTA esta descrito por summary/error;
- `courseError` queda en `InlineMessage` con id;
- `resolveActiveRoute.jsx` conserva `course.openCourses` con `answers`, `assessment` y `authToken`.
