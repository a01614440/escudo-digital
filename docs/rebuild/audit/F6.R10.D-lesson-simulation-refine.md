# F6.R10.D — Lesson + simulaciones refine

Fecha: 2026-04-24

## Objetivo

Cerrar el frente visual de lesson/stage/simulaciones despues del hard rebuild de `Mi ruta`, sin tocar dominio ni el renderer.

El problema principal era que varias superficies aun conservaban una logica de `stage + rail` heredada: en desktop se generaban columnas de soporte, rails sticky y grids que seguian compitiendo con la practica. Eso era aceptable como cierre tecnico de F6.R9, pero no como cierre visual para presentacion.

## Cambios implementados

### Lesson

- `LessonView.jsx` agrega `data-sd-r10d-lesson="stage-recovery"` al root.
- El bloque secundario de lesson mantiene su rol subordinado, pero ahora usa `data-sd-r10d-secondary="stacked"`.
- Se elimino la composicion desktop `md:grid-cols-2 xl:grid-cols[...]` del bloque secundario.
- Resultado esperado: stage primero, contexto debajo, sin paneles laterales compitiendo.

### Activity chrome

- `ActivityChrome` agrega `data-sd-r10d-stage="dominant"` al wrapper inmersivo.
- Ese marcador activa el contrato R10.D para simulaciones: stage dominante, soportes subordinados y rails sin sticky.

### Simulaciones especificas

- `InboxActivity.jsx`
  - Agrega `data-sd-r10d-simulation="list-stack"`.
  - El stage `list-detail` agrega `data-sd-r10d-stage="single-column"`.
  - Se elimina el `xl:grid-cols[...]` que forzaba lista + detalle como split rigido.

- `WebLabActivity.jsx`
  - El briefing agrega `data-sd-r10d-briefing="stacked"`.
  - El workbench agrega `data-sd-r10d-stage="single-column"`.
  - Se eliminan los grids `xl:grid-cols[...]` del briefing y del workbench.

- `ScenarioFlowActivity.jsx`
  - El stage agrega `data-sd-r10d-stage="single-column"`.
  - Se elimina el split desktop `xl:grid-cols[...]`.

### CSS

- `tailwind.css` agrega reglas R10.D para:
  - contextos secundarios de lesson en una sola columna;
  - simulaciones inmersivas con `--sd-simulation-stage-max-inline: 110rem`;
  - `.sd-simulation-main-stage` y `.sd-chat-sim-desktop` en una sola columna cuando el wrapper R10.D esta activo;
  - rails subordinados con `position: static`, `max-height: none` y `overflow: visible`;
  - texto dentro del stage con `word-break: normal` y `overflow-wrap: normal`.

## Causa raiz corregida

El problema no era un solo componente. Persistian tres causas combinadas:

1. `LessonView` aun tenia una grilla secundaria de dos columnas en desktop.
2. Las simulaciones mantenian `xl:grid-cols[...]` en JSX para inbox, WebLab y ScenarioFlow.
3. El CSS global hacia sticky cualquier `[data-sd-stage-rail="subordinate"]` en desktop.

R10.D no borra la semantica de soporte, pero cambia su comportamiento: soporte sigue existiendo, solo que deja de aplastar la practica.
