# F6.R4 - Cursos / modulos convergence

Fecha: 2026-04-23
Estado: cerrada localmente

## Objetivo

Reducir la sensacion de ruta partida entre rail, detalle, metadata y lista de actividades:

- menos lectura administrativa;
- menos bloques hermanos compitiendo;
- un mission board mas convergente;
- mejor relacion entre modulo activo, progreso y CTA;
- lista de actividades mas legible.

## Cambios implementados

### 1. Mission board convergente

Archivo: `frontend/src/components/CoursesView.jsx`

- `ModuleMissionBoard` deja de depender de `ProgressSummary` y de metadata suelta en bloques separados.
- Se crea un `sd-module-command-deck` unico con:
  - siguiente bloque;
  - progreso principal;
  - facts de apoyo;
  - nota de estado si aplica;
  - CTA principal.
- La categoria del modulo pasa a convivir con el estado recomendado en el encabezado.
- El objetivo queda resumido en una sola linea de apoyo, sin tono de microcopy tecnico.

Resultado:

- el detalle del modulo ya no se siente como progreso + facts + CTA repartidos;
- la accion principal cae dentro del mismo bloque donde se entiende el contexto.

### 2. Facts de apoyo mas utiles

Archivo: `frontend/src/components/CoursesView.jsx`

- Se sustituyen stats secundarias dispersas por tres facts claras:
  - `Categoria`
  - `Gap visible`
  - `Siguiente desbloqueo`
- Estas facts viven en una sola grilla compacta (`sd-module-support-facts`) dentro del command deck.

Resultado:

- menos paneles secundarios;
- menos lectura en zig-zag entre surfaces hermanas;
- mejor continuidad entre decision y accion.

### 3. Mapa de actividades mas legible

Archivo: `frontend/src/components/CoursesView.jsx`

- `ModuleActivityList` deja la composicion rigida tipo admin.
- Cada fila ahora usa:
  - bloque flexible;
  - titulo con `break-words`;
  - badge semantico por estado;
  - highlight mas visible para la siguiente actividad.
- El toggle pasa de `Ver actividades del modulo` a `Mapa de actividades`, con un badge corto de avance.

Resultado:

- mejor lectura de modulos largos;
- menos sensacion de tabla o checklist administrativo;
- mejor foco sobre la actividad que sigue.

### 4. Guard especifico de la subfase

Archivo: `test/f6-module-convergence-smoke.test.js`

Protege:

- que el rail siga siendo secundario;
- que el mission board mantenga el command deck convergente;
- que los facts de categoria/gap/desbloqueo sigan agrupados;
- que la lista de actividades no vuelva al grid rigido anterior.

### 5. Guards historicos alineados

Archivo: `test/f4-courses-dashboard-smoke.test.js`

- Se actualizan aserciones que todavia esperaban `ProgressSummary`, el summary viejo del toggle y el highlight anterior de actividades.
- Los guards ahora fijan la forma buena del mission board convergente.
