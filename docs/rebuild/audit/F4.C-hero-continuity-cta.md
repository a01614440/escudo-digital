# F4.C - Hero + continuity + CTA pass

Fecha: 2026-04-23

## Objetivo

Fortalecer la entrada principal de `CoursesView` despues del pase de densidad F4.B.

F4.C se limito a:

- hacer que el hero oriente sin competir con la accion;
- convertir `ContinuityConsole` en la region dominante de decision;
- mover el CTA principal antes del progreso de apoyo;
- mejorar contraste y jerarquia usando foundation existente;
- mantener intactos continuidad, modulo seleccionado, progreso y dominio.

## Cambios aplicados

### RouteHero

- El titulo paso a `Tu ruta ya esta lista para continuar.`
- El subtitulo usa `prioritySummary` en vez de `assessment?.resumen`.
- El hero queda como orientacion de ruta y foco, no como segundo centro de accion.
- No se agrego CTA dentro del hero para evitar competir con `ContinuityConsole`.
- Se mantiene el badge admin cuando aplica.

### ContinuityConsole

- La superficie cambio a `SurfaceCard variant="command" tone="inverse"` para darle jerarquia visual real usando foundation.
- La region ahora se lee como el centro de decision de la pantalla.
- El CTA primario aparece antes que el progreso.
- El CTA primario usa `data-sd-primary-cta="courses-continuity"` para que la guarda de F4.C detecte regresiones.
- El label se simplifico:
  - `Continuar mi ruta` cuando ya hay avance;
  - `Abrir modulo recomendado` cuando no hay avance.
- Se agrego `aria-label` con el titulo del modulo para que el CTA sea mas especifico sin meter mas copy visible.
- El secundario bajo a `Ver en la ruta`.
- `ProgressSummary` queda despues del CTA como apoyo, no como elemento dominante.
