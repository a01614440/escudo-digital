# F5.G - Comfort / jugabilidad / responsiveness pass

Fecha: 2026-04-23

## Objetivo

Cerrar el ultimo pase de implementacion de F5 expandida antes de la validacion final.

Esta subfase no redisenia simulaciones ni toca dominio. Ajusta proporciones, contexto secundario, contraste y hooks responsivos para que la experiencia ruta -> lesson -> practica se sienta mas comoda y menos dashboard-first.

## Cambios cerrados

### 1. Ruta mas balanceada

`CoursesView` conserva el layout de dos paneles, pero baja la agresividad de las proporciones:

- tablet: `minmax(17rem,19rem) + minmax(0,1fr)`;
- desktop: `minmax(18rem,20rem) + minmax(0,1fr)`;
- `data-sd-route-comfort="balanced-two-pane"`.

Esto evita que el detalle compita contra un rail demasiado ancho o contra ratios inflados.

### 2. Stepper como contexto secundario

`DashboardSceneBar` mantiene `JourneyStepper`, pero ya no lo presenta como una fila permanente de peso completo.

Ahora vive en:

- `details.sd-dashboard-stepper-toggle`;
- `data-sd-journey-stepper="courses-route"`;
- summary `Ver progreso de la ruta`.

El progreso queda disponible sin seguir ocupando aire vertical si no hace falta.

### 3. Lesson guiado con menos envoltura

`LessonActivityStage` baja a `padding="md"` en todos los shells y marca:

- `data-sd-stage-comfort="compact"`.

El renderer frame tambien baja de padding/radio para recuperar espacio util sin tocar `ActivityRenderer` ni actividades internas.

### 4. Mapa de modulo secundario

`LessonCommandRail` deja el mapa completo como contexto colapsable en todos los shells, no solo en mobile.

El mapa queda marcado con:

- `data-sd-lesson-map="secondary"`.

Esto evita que el recorrido del modulo robe atencion o ancho cuando el usuario ya esta practicando.

### 5. Fullscreen feeling para inmersivas

`tailwind.css` refuerza:

- ancho completo de `.sd-lesson-layout-immersive .sd-lesson-primary`;
- `min-height: clamp(34rem, 72vh, 58rem)` en `.sd-immersive-activity-shell`.

No cambia el look interno de WhatsApp, Inbox, WebLab, CallSimulation ni ScenarioFlow. Solo garantiza que el contenedor comun les da presencia.

### 6. Contraste y lectura final

`tailwind.css` suma hooks de contraste para:

- `.sd-route-pill[aria-current='true']`;
- `.sd-route-pill[data-route-module-state='recommended']`;
- titulos display de route/lesson con ancho legible.
