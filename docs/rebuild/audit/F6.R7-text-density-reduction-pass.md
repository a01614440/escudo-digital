# F6.R7 - Text density reduction pass

Fecha: 2026-04-24
Estado: cerrada localmente

## Objetivo

Reducir texto visible simultaneo en Survey, Mi ruta, Lesson y simulaciones sin cambiar dominio, scoring, contratos ni flujos.

Esta subfase parte de F6.R6: el contraste ya estaba endurecido, pero todavia habia demasiada explicacion abierta al mismo tiempo. El objetivo fue que cada pantalla diga menos, priorice mejor y esconda contexto secundario cuando no es necesario para actuar.

## Cambios implementados

### 1. Feedback mas accionable y menos largo

Archivo: `frontend/src/components/FeedbackPanel.jsx`

- El resumen principal queda limitado a dos datos visibles:
  - `Que viste`
  - `Accion real`
- La informacion secundaria paso a un bloque collapsable:
  - `Que falto`
  - `Siguiente paso`
  - senales detectadas
  - senales faltantes
- El subtitulo del feedback estructurado paso de `Criterio y siguiente accion.` a `Resultado breve.`
- Los mensajes finales quedaron mas cortos:
  - `Conserva esa rutina.`
  - `Revisa el ajuste antes de avanzar.`
  - `Afina la rutina y sigue.`

Resultado:

- feedback conserva lo necesario para decidir;
- el detalle pedagogico sigue disponible sin invadir la pantalla;
- las simulaciones pierden menos protagonismo al cerrar una respuesta.

### 2. Summary bars con maximo dos hechos visibles

Archivo: `frontend/src/components/activities/sharedActivityUi.jsx`

- `ActivitySummaryBar` ahora acepta `maxItems = 2`.
- La barra filtra y corta con `.slice(0, maxItems)`.
- Las captions quedan ocultas por defecto con `showCaptions = false`.

Resultado:

- las simulaciones dejan de abrir con demasiadas micro-metricas;
- el stage mantiene contexto breve sin convertirse en panel informativo.

### 3. Survey mas sintetico

Archivo: `frontend/src/components/SurveyView.jsx`

- Intro y panel inicial reducen copy visible.
- Los badges de canales se limitan a tres.
- El progreso ya no muestra la frase redundante `Puedes volver atras sin perder respuestas.`
- La ayuda contextual usa `Lo minimo para decidir.`
- Loading reduce la lista visible a dos pasos.
- Results reduce recomendaciones a dos y siguientes pasos a uno.
- El handoff a ruta usa copy corto: `Perfil listo; abre tu ruta.`

Resultado:

- Survey se entiende mas rapido;
- el resultado deja de parecer una pantalla de lectura larga;
- el CTA hacia ruta queda mas limpio.

### 4. Mi ruta y cursos reducen contexto secundario

Archivo: `frontend/src/components/CoursesView.jsx`

- El estado de generacion usa copy mas corto.
- La linea de continuidad superior solo muestra el siguiente hito real.
- `RouteNavigatorRail` usa `Cambia de modulo sin perder foco.`
- El filtro de nivel pierde descripcion redundante.
- El objetivo del modulo se mueve a `details`.
- Los facts de soporte se limitan a dos visibles.
- Progreso y ajustes recortan subtitulos y textos de ayuda.

Resultado:

- Mi ruta se siente menos fragmentada;
- la lista y el modulo activo compiten menos con texto secundario;
- el usuario ve antes el siguiente paso.

### 5. Lesson reduce briefing redundante

Archivo: `frontend/src/components/LessonView.jsx`

- Se elimino copy duplicado bajo el titulo de actividad.
- La linea de metadata se redujo a categoria y nivel.
- Se retiro el parrafo visible del guided stage que repetia intro/prompt.
- El rail de insight perdio subtitulo y cambio bloques largos por badges compactos.
- El objetivo pedagogico paso a `details`.
- Completion retiro el objetivo largo del modulo.

Resultado:

- la practica gana protagonismo;
- el contexto queda disponible, pero subordinado;
- el stage se siente menos atrapado por explicaciones.

### 6. Simulaciones con menos paneles largos

Archivos:

- `frontend/src/components/activities/signalActivities.jsx`
- `frontend/src/components/activities/immersive/InboxActivity.jsx`
- `frontend/src/components/activities/immersive/WebLabActivity.jsx`
- `frontend/src/components/activities/immersive/ScenarioFlowActivity.jsx`
- `frontend/src/components/activities/CallSimulationActivity.jsx`

Cambios:

- Chat/signal limita listas de apoyo a dos senales.
- Inbox/SMS/email elimina parrafos introductorios redundantes y limita review a tres items.
- WebLab acorta briefing, ayudas, hallazgos, decision y review.
- ScenarioFlow reduce intro, rutina segura y copy de review.
- CallSimulation acorta permisos/modos y mueve transcript preview a `details`.

Resultado:

- menos texto visible simultaneo;
- menos paneles compitiendo con la simulacion;
- los detalles largos quedan bajo demanda.

### 7. Estilos para detalle secundario collapsable

Archivo: `frontend/src/styles/tailwind.css`

- Se agregaron estilos para:
  - `.sd-feedback-secondary`
  - `.sd-feedback-secondary summary`
  - grids internos de detalle y senales

Resultado:

- el detalle secundario del feedback queda visualmente subordinado;
- no se pierde informacion, pero deja de abrirse como texto obligatorio.
