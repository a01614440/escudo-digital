# F3.E - Survey flow hardening

Fecha: 2026-04-23

## Objetivo

Cerrar el hardening minimo del flujo visual de `SurveyView.jsx`:

- intro;
- pregunta actual;
- navegacion interna;
- validacion visible;
- loading visible y anunciado;
- continuidad del render entre escenas.

No se cambio `useAssessmentFlow.js`, no se toco scoring, no se toco `requestAssessment`, no se cambio results/CTA y no se abrio F3.F.

## Cambios realizados

### 1. Gate explicito para la intro

Archivo:

- `frontend/src/components/SurveyView.jsx`

Se agrego `shouldShowSurveyIntro` para centralizar la condicion de entrada a la intro:

- sin assessment;
- `surveyStage === 'survey'`;
- `surveyIndex === 0`;
- sin respuestas.

Esto evita que la condicion viva duplicada dentro de `useState` y render condicional.

### 2. Reset disciplinado de intro tras reinicio real

Archivo:

- `frontend/src/components/SurveyView.jsx`

Se agrego `introResetPendingRef` para distinguir:

- volver desde fuera de `survey` o desde results/loading;
- versus simplemente borrar una respuesta en la primera pregunta.

El comportamiento queda asi:

- usuario nuevo: ve intro;
- usuario pulsa comenzar: la intro se oculta;
- usuario responde: la intro queda oculta;
- request falla y vuelve de loading a survey con respuestas: la intro no reaparece;
- usuario reinicia desde results y el hook vuelve a `survey`, index `0`, sin respuestas ni assessment: la intro vuelve a aparecer.

Esto cierra el riesgo detectado en F3.A sin tocar `resetSurveyState` ni `useAssessmentFlow`.

### 3. Resolucion unica de escena activa

Archivo:

- `frontend/src/components/SurveyView.jsx`

Se agrego `getSurveyScene` y `activeSurveyScene`.

Antes, las escenas se renderizaban con condiciones independientes:

- `showIntro`;
- `surveyStage === 'survey' && !showIntro`;
- `surveyStage === 'loading'`;
- `surveyStage === 'results'`.

Ahora el render depende de una sola escena:

- `intro`;
- `survey`;
- `loading`;
- `results`.

Esto evita renders superpuestos si `showIntro` queda true mientras `surveyStage` cambia.

### 4. Loading anunciado como estado ocupado

Archivo:

- `frontend/src/components/SurveyView.jsx`

El panel principal de loading ahora usa:

- `role="status"`;
- `aria-live="polite"`;
- `aria-busy="true"`;
- `SurfaceCard tone="inverse"`;
- tokens tipograficos `sd-heading-sm` y `sd-copy-sm`.

Se retiraron hacks locales del loading:

- shadow arbitrario `shadow-[0_30px_80px...]`;
- overrides `[&_.text-sd-text]:text-white`;
- `text-white/76`.

### 5. Guards source-level agregados

Archivo:

- `test/f3-auth-survey-smoke.test.js`

Cobertura agregada:

- existe `shouldShowSurveyIntro`;
- existe `getSurveyScene`;
- existe `introResetPendingRef`;
- el reset de intro se activa al salir de survey/results/loading;
- la intro vuelve solo si `canShowIntro` y el reset esta pendiente;
- el pending se limpia si ya no aplica mostrar intro;
- el render usa `activeSurveyScene`;
- loading usa `role="status"`, `aria-live`, `aria-busy`;
- loading ya no usa hacks inversos locales.
