# F4.D - Route navigator + module detail pass

Fecha: 2026-04-23

## Objetivo

Cerrar el pase de F4 dedicado al navegador de ruta, lectura de modulos y detalle seleccionado dentro de `CoursesView.jsx`.

F4.D se limito a:

- bajar el peso visual del rail de modulos;
- mejorar escaneo y wrapping de cada modulo;
- hacer mas explicito el estado seleccionado/bloqueado/recomendado;
- conectar lista -> detalle con atributos estables;
- hacer que el detalle del modulo responda mas rapido que sigue y como abrirlo;
- mantener fuera lesson shell, activity chrome, simulaciones, progreso/stats globales y layout comfort.

## Cambios aplicados

### RouteNavigatorRail

- El rail bajo de `variant="command"` con fondo inverso local a `variant="support"`.
- Se retiraron hacks visuales locales:
  - gradiente hardcoded;
  - shadow arbitrario;
  - overrides de texto blanco.
- El titulo paso a `Ruta por modulos`.
- El subtitulo ahora comunica la tarea real: escanear estado, prioridad y detalle seleccionado.
- El filtro de nivel usa `SurfaceCard variant="subtle"` en vez de una superficie oscura local.

### RouteModulePill

- Se agregaron labels derivados para categoria, nivel y estado.
- Se agrego `aria-current` cuando el modulo esta seleccionado.
- Se agrego `aria-label` especifico con titulo, estado, categoria y nivel.
- Se agrego `data-route-module-state` para distinguir `locked`, `recommended` y estados de progreso.
- Se retiro el shadow arbitrario del seleccionado.
- El titulo del modulo usa `break-words` y leading mas comodo.
- El estado ahora se muestra como `Badge` semantico, no como texto suelto.
- El metadato visible categoria/nivel queda en una linea limpia.

### ModuleActivityList

- La actividad siguiente se marca con `aria-current="step"`.
- Cada actividad expone `data-activity-state` (`completed`, `next`, `pending`).
- Se retiro tracking manual del indice interno.
- Las cards internas usan radius mas contenido.

### ModuleMissionBoard

- El contenedor del detalle expone `data-route-detail="module"` y `data-selected-module-id`.
- El titulo del modulo se calcula una vez como `moduleTitle`.
- La siguiente actividad se calcula una vez como `nextActivityTitle`.
- El copy fallback del hero se acorto.
- El panel de ruta interna ahora prioriza la actividad siguiente antes de la secuencia.
- El rail de apertura se simplifico a accion directa para el modulo seleccionado.
- El CTA local aparece antes de `ProgressSummary`.
- El CTA local expone `data-sd-module-cta="courses-detail"` y `aria-label` con el titulo del modulo.
- Se retiro el `KeyValueBlock` de `Ultimo cierre` / `Actividades`, porque repetia progreso sin ayudar a decidir.

## Guardas agregadas

Se amplio `test/f4-courses-dashboard-smoke.test.js` con guardas F4.D:

- `RouteNavigatorRail` debe usar `variant="support"` y no debe recuperar hacks inversos locales.
- `RouteModulePill` debe exponer `aria-current`, `aria-label`, `data-route-module-state`, wrapping y badge semantico.
- `ModuleActivityList` debe marcar la actividad siguiente con `aria-current="step"` y `data-activity-state`.
- `ModuleMissionBoard` debe conectar detalle seleccionado y mostrar el CTA local antes del progreso de apoyo.
- `ModuleMissionBoard` no debe recuperar `KeyValueBlock` ni `Ultimo cierre`.
