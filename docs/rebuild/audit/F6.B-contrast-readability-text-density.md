# F6.B - Contrast / readability / text-density system pass

Fecha: 2026-04-23
Commit base: `4879458` (`audit(F6.A): simulation baseline global`)

## 1. Objetivo

Cerrar el primer pase sistemico de F6: subir legibilidad y reducir densidad visual en las piezas compartidas que rodean a las simulaciones, sin redisenar canales especificos todavia.

F6.B no intenta resolver WhatsApp, SMS, Inbox, WebLab, CallSimulation o ScenarioFlow por separado. Eso empieza en F6.C-F6.I. Esta subfase deja una base comun para que las simulaciones no vuelvan a apoyarse en superficies translucidas, copy secundario lavado o summary cards demasiado flojas.

## 2. Cambios realizados

### Activity chrome / briefing / guide

Archivo: `frontend/src/components/activities/sharedActivityUi.jsx`

- `SimulationGuide` ahora usa `sd-simulation-readable-surface`.
- `SimulationGuide` declara `data-sd-text-density="compact"`.
- Los pasos de guia se movieron a clases reutilizables:
  - `sd-simulation-guide-list`
  - `sd-simulation-guide-step`
  - `sd-simulation-guide-index`
- `ActivityChrome` usa la misma superficie F6 para el briefing guiado.
- `ActivityChrome` marca el briefing con `data-sd-text-density="compact"`.

Resultado: briefing y guia ya no dependen de `bg-white/84` ni de bloques sueltos con contraste variable.

### ActivitySummaryBar

Archivo: `frontend/src/components/activities/sharedActivityUi.jsx`

- `ActivitySummaryBar` ahora declara `data-sd-activity-summary="compact"`.
- Las cards internas usan `SurfaceCard variant="support"` en lugar de `variant="subtle"`.
- Se agregaron clases dedicadas:
  - `sd-activity-summary-grid`
  - `sd-activity-summary-card`
  - `sd-activity-summary-label`
  - `sd-activity-summary-value`
  - `sd-activity-summary-caption`
- Se retiro el tracking manual `tracking-[0.14em]` del label del summary.
- La caption ya no queda como texto demasiado suave.

Resultado: las barras de resumen se sienten mas compactas y legibles sin cambiar la informacion funcional.

### FeedbackPanel

Archivo: `frontend/src/components/FeedbackPanel.jsx`

- `FeedbackPanel` declara `data-sd-text-density="compact"`.
- El subtitulo del feedback string se redujo a: `Resumen corto para decidir el siguiente paso.`
- El subtitulo del feedback estructurado se redujo a: `Criterio y siguiente accion.`

Resultado: el feedback conserva su estructura F5.F, pero deja de meter copy extra que compite con la practica.

### Immersive primitives

Archivos:

- `frontend/src/components/activities/immersive/shared.js`
- `frontend/src/components/activities/immersive/immersivePrimitives.jsx`

Cambios:

- `PANEL_CLASS` ahora incluye `sd-immersive-panel`, `border-sd-border-strong` y `bg-sd-surface-raised`.
- `SOFT_PANEL_CLASS` ahora incluye `sd-immersive-aside-panel`, `border-sd-border-strong` y `bg-sd-surface`.
- Se retiraron washes `bg-white/75` y `bg-white/60` de las primitives inmersivas compartidas.
- `ImmersiveAsidePanel` usa `text-sd-text-soft` en vez de `text-sd-muted`.
- `ImmersiveProgressPill` ahora usa clases dedicadas:
  - `sd-immersive-progress-pill`
  - `sd-immersive-progress-label`
- Se retiro `bg-white/76` y `text-sd-muted` del progress pill.

Resultado: panels y pills compartidos tienen mejor contraste y se apoyan en tokens adaptables, no en blancos translucidos.

### Tailwind layer

Archivo: `frontend/src/styles/tailwind.css`

Se agregaron o reforzaron hooks F6:

- `sd-simulation-readable-surface`

- `sd-activity-summary-grid`

- `sd-activity-summary-value`

- `sd-immersive-panel`

Tambien se reforzo:

- `sd-feedback-copy` a `text-sd-text`;
- `sd-feedback-item` / `sd-feedback-list` con `border-sd-border-strong`, `bg-sd-surface` y cuerpo `text-sd-text`.
