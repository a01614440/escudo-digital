# F6.R5 - Lesson + simulation shell redesign

Fecha: 2026-04-23
Estado: cerrada localmente

## Objetivo

Rearmar `LessonView` para que la practica deje de sentirse rail-first y pase a una lectura realmente content-first:

- stage primero;
- contexto secundario debajo;
- menos paneles simultaneos;
- empty/completion sin `SplitHeroLayout`;
- guided stage mas dominante;
- secondary region mas resumida y menos dashboardera.

## Cambios implementados

### 1. Empty y completion salen del lenguaje split-hero

Archivo: `frontend/src/components/LessonView.jsx`

- `ModuleEmptyState` ya no usa `SplitHeroLayout`.
- `ModuleComplete` ya no usa `SplitHeroLayout`.
- Ambos pasan a un `sd-lesson-shell-command` sobre `SurfaceCard variant="command" tone="inverse"`.
- El estado, CTA principal y continuidad viven ahora dentro del mismo bloque fuerte.

Resultado:

- entry/exit del lesson ya no se sienten como otra vista heredada del dashboard;
- el shell habla el mismo idioma visual que la entrada a practica.

### 2. Stage-first real en el lesson

Archivo: `frontend/src/components/LessonView.jsx`

- `LessonView` deja la logica `guided-two-pane` / `immersive-fullscreen` con rail lateral persistente.
- La nueva lectura es:
  - hero;
  - stage dominante;
  - secundarios subordinados debajo.
- El root ahora expone:
  - `data-sd-lesson-layout={isImmersive ? 'immersive-stack' : 'guided-stack'}`
  - `data-sd-lesson-comfort="stage-first"`
  - `data-sd-lesson-secondary="subordinate"`

Resultado:

- la practica manda antes que el contexto;
- el shell deja de comprimir el stage por costumbre.

### 3. Command rail e insight rail dejan de ser rails de verdad

Archivo: `frontend/src/components/LessonView.jsx`

- `LessonCommandRail` deja `SupportRail` + `ProgressSummary`.
- Ahora es una `SurfaceCard` compacta con:
  - nombre del modulo;
  - progreso corto;
  - `ProgressBar`;
  - mapa colapsable;
  - acciones.
- `LessonInsightRail` deja `SupportRail` y se reduce a una `SurfaceCard` breve con:
  - `KeyValueBlock`;
  - objetivo;
  - lectura del modulo colapsable;
  - ultima reentrada solo como nota corta.

Resultado:

- menos chrome visible al mismo tiempo;
- secundarios mas resumidos;
- menos sensacion de rail administrativo.

### 4. Guided stage mas dominante

Archivo: `frontend/src/components/LessonView.jsx`

- `LessonActivityStage` guided ya no usa `SurfaceCard` + `PanelHeader`.
- El stage ahora es una superficie directa con header corto y renderer frame dominante.
- `data-sd-stage-comfort` pasa a:
  - `dominant` en guided
  - `fullscreen` en immersive

Resultado:

- guided deja de sentirse como practica metida dentro de otra card;
- el renderer gana peso real.

### 5. Hooks visuales nuevos en Tailwind

Archivo: `frontend/src/styles/tailwind.css`

Se agregaron o ajustaron hooks para:

- `sd-lesson-flow`
- `sd-lesson-secondary-grid`
- `sd-lesson-shell-command`
- `sd-lesson-shell-status`
- `sd-lesson-support-card`
- `sd-lesson-insight-toggle`
- `sd-lesson-stage-guided[data-sd-stage-comfort='dominant']`

Resultado:

- el lesson mantiene una jerarquia mas clara sin depender de grids laterales rigidos.

### 6. Guards nuevos y actualizados

Archivos:
