# F2.B - DeviceShell dispatcher hardening

Fecha: 2026-04-23

## Objetivo

Cerrar el bug silencioso detectado en F0.4/F2.A: `DeviceShell` invocaba shells como funciones en vez de renderizarlos como componentes React.

## Cambio realizado

Archivo modificado:

- `frontend/src/shells/DeviceShell.jsx`

Antes:

```js
const renderShell = SHELL_RENDERERS[shellFamily] || SHELL_RENDERERS.desktop;
return renderShell({ ... });
```

Ahora:

```jsx
const Shell = SHELL_RENDERERS[shellFamily] || SHELL_RENDERERS.desktop;
return <Shell ... />;
```

## Por que importa

El patron anterior funcionaba solo porque `MobileShell`, `TabletShell` y `DesktopShell` no usan hooks. Si algun shell agregaba hooks despues, la llamada como funcion podia fusionar esos hooks con `DeviceShell` y romper el orden de hooks al cambiar `shellFamily`.

El nuevo dispatcher mantiene cada shell como componente React real.

## Fuera de alcance respetado

- No se tocaron `MobileShell`, `TabletShell` ni `DesktopShell`.
- No se tocaron vistas grandes.
- No se tocaron route-containers.
- No se toco `App.jsx`.
- No se tocaron hooks de dominio.
- No se toco backend, services, DB, scoring, `ActivityRenderer` ni `activityRegistry`.
