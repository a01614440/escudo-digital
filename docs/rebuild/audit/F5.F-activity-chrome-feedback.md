# F5.F - Activity chrome + instructions + feedback integration

Fecha: 2026-04-23

## Objetivo

Cerrar la integracion comun de instrucciones, activity chrome y feedback sin invadir el diseno interno de simulaciones especificas.

Esta subfase absorbe tres problemas detectados en F5.A:

- el lesson stage repetia briefing que ya podia pertenecer al activity chrome;
- las instrucciones y guias ocupaban demasiado espacio permanente;
- el feedback se presentaba como tarjetas dentro de tarjetas, agregando peso visual despues de cada intento.

## Cambios cerrados

### 1. Lesson stage delega el briefing visible

`LessonActivityStage` mantiene la superficie de actividad guiada y el renderer, pero ya no renderiza los dos bloques locales de instrucciones.

Ahora el stage queda marcado con:

- `data-sd-lesson-stage="guided"`
- `data-sd-briefing-source="activity-chrome"`

La fuente visible del briefing queda en `ActivityChrome`, no duplicada en `LessonView`.

### 2. ActivityChrome como fuente unica del briefing guiado

`ActivityChrome` conserva el bypass inmersivo para:

- `sim_chat`
- `inbox`
- `web_lab`
- `call_sim`
- `scenario_flow`

Para actividades guiadas, ahora expone:

- `data-sd-activity-chrome="guided"`
- `data-sd-briefing="activity-chrome"`
- un briefing colapsable con titulo, quick tip, categoria, nivel y dos criterios esenciales;
- el renderer de la actividad debajo, sin tarjetas de instrucciones anidadas.

Esto evita que la practica se sienta atrapada entre varias superficies secundarias.

### 3. SimulationGuide queda secundaria y colapsable

`SimulationGuide` ya no alterna entre una version en card y una version compacta.

Ahora siempre es un `details` colapsable con:

- `data-sd-simulation-guide="collapsed"`
- pasos visibles solo si el usuario decide abrirlos;
- menor peso visual permanente.

### 4. FeedbackPanel queda compacto

`FeedbackPanel` conserva:

- lectura de tono por score/titulo;
- `InlineMessage` accesible;
- score badge;
- secciones de senales detectadas y faltantes.

Pero elimina los `SurfaceCard` internos para cada bloque. La retroalimentacion queda en una sola superficie con secciones ligeras:

- `sd-feedback-panel`
- `sd-feedback-summary-grid`
- `sd-feedback-signal-grid`
- `sd-feedback-item`
- `sd-feedback-list`

## Fuera de alcance respetado

No se tocaron:

- simulaciones especificas;
- `ActivityRenderer.jsx`;
- `activityRegistry.js`;
- hooks de dominio;
- backend;
- services;
- scoring;
- contracts;
- `CoursesView.jsx`.

## Resultado

F5.F deja las instrucciones y el feedback mas integrados al flujo de practica. La actividad conserva el protagonismo, mientras el contexto secundario se vuelve consultable y menos invasivo.
