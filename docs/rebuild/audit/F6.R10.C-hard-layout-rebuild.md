## Causa raiz del texto vertical

La causa no era un solo `word-break`.

El texto vertical venia de la combinacion de:

1. `RouteNavigatorRail` todavia funcionaba como rail/lista secundaria con tarjetas demasiado estrechas.
2. La lista de modulos usaba grids responsive (`sm:grid-cols-2`, `xl:grid-cols-3`) que podian reducir demasiado el ancho de cada card.
3. `RouteModulePill` ponia el titulo y el badge de estado en la misma fila flex. El titulo tenia `min-w-0 flex-1`; cuando el contenedor se estrechaba, quedaba con ancho casi nulo.
4. La clase legacy global `.panel { overflow-wrap: anywhere; }` seguia siendo peligrosa para superficies que quedaban estrechas.
5. `RouteBriefing` tenia `data-sd-container="true"` pero solo `max-width`; por containment, el card podia encogerse al ancho del contenido/padding. Eso generaba el hero comprimido y mucho espacio muerto alrededor.

Correccion aplicada:

- `RouteModulePill` separa titulo y badges en bloques verticales.
- Los titulos usan `word-break: normal`, `overflow-wrap: normal`, `hyphens: none`, `white-space: normal`.
- El navegador de modulos queda como lista amplia de una sola columna.
- El layout de ruta usa `hard-stack` y `data-sd-route-comfort="hard-rebuild"`.
- `RouteBriefing` ahora tiene ancho explicito `width: min(100%, 80rem)` y su grid interno ocupa `width: 100%`.
- Los tabs del shelf de ruta ya no usan `variant="hero"` sobre fondo claro; el activo usa `primary`.
